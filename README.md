# StyledMD v2

Windows 전용 마크다운 에디터. Go 백엔드와 WebView2(Chromium) 기반 프론트엔드로 구성된 단일 실행 파일(`StyledMD.exe`)로 작동한다.

---

## 1. 앱소개

### 1.1 앱의 목적
이 애플리케이션은 복잡한 설치 과정 없이 단일 실행 파일(`StyledMD.exe`) 형태로 실행되는 가볍고 직관적인 Windows 전용 마크다운 에디터다. 사용자가 작성하는 문서 정보를 로컬에서 실시간으로 확인하고 제어하며, 다채로운 AI 엔진들과의 간편한 연동을 제공하여 문서 생성 및 다듬기 과정을 극대화하는 것을 목적으로 한다.

### 1.2 앱의 특징과 스크린샷

![StyledMD Screenshot](./StyledMD_ScreenShot.jpg)

* **실시간 렌더링**: 에디터 영역(Monaco Editor)에서 문서를 입력하는 즉시 오른쪽 화면에 렌더링 결과(marked.js)가 동기화되어 나타난다.
* **로컬 폐쇄형 작동**: 외부 서버의 개입 없이 로컬 포트를 자동 할당받아 내부 WebView2 창과 HTTP API 통신을 수행한다.
* **보안 암호화**: 설정된 API 키들은 Windows DPAPI 및 AES-256-GCM 알고리즘으로 이중 암호화되어 `%USERPROFILE%\Documents\.apikeys.json`에 완전하게 보존된다.

#### 1.2.1 마크다운 태그의 추가

##### 1.2.1.1 선택된 텍스트에 색을 넣기
특정 텍스트 영역에 글자색과 배경색을 지정할 수 있는 인라인 확장 문법을 지원한다.
* **사용법**: `[텍스트]{color: 글자색 bg: 배경색}` 또는 `[텍스트]{글자색 bg: 배경색}`
* **예시**:
  ```markdown
  [이 텍스트는 빨간색 글자에 노란색 배경이다]{color: red bg: yellow}
  [이 텍스트는 파란색 글자다]{blue}
  [이 텍스트는 배경만 분홍색이다]{bg: pink}
  ```

##### 1.2.1.2 callout 사용법
문서 내에서 특정 문장을 강조하기 위한 5가지 테마의 Callout 문법을 지원한다.
* **지원 종류**: `IMPORTANT`, `NOTE`, `TIP`, `WARNING`, `QUESTION`
* **사용법**:
  ```markdown
  > [!NOTE]
  > 이것은 일반적인 노트 메시지다.
  
  > [!TIP]
  > 꿀팁이나 최적화 팁을 여기에 작성한다.
  
  > [!WARNING]
  > 경고나 주의가 필요한 내용을 적는다.
  
  > [!QUESTION]
  > 질문사항이나 의문을 표시한다.
  
  > [!IMPORTANT]
  > 필수적으로 알아야 할 중요 항목이다.
  ```

##### 1.2.1.3 빈줄 넣는 ||
기존 마크다운은 여러 빈 줄을 표현하기 까다롭지만, 줄의 시작 부분에 `||` 문자를 입력하면 즉각적인 개행(`<br>`)으로 렌더링된다.
* **사용법**:
  ```markdown
  첫 번째 문장입니다.
  ||
  한 줄을 띄우고 두 번째 문장입니다.
  ||
  ||
  두 줄을 띄우고 세 번째 문장입니다.
  ```

#### 1.2.2 인공지능 팝업창

##### 1.2.2.1 API Key와 서비스 모델을 설정하기
애플리케이션 우측 상단의 설정 아이콘이나 API Key 관리 메뉴를 클릭하여 팝업창을 호출한 후 설정한다.
* **제공 API**: Google API, Groq API, Ollama API, Upstage API
* **저장 위치**: `%USERPROFILE%\Documents\.apikeys.json` (이중 암호화 적용)
* 각 API 제공업체의 키와 모델 이름(예: `gemini-2.5-flash`), 그리고 UI에 표시될 모델 별칭(Alias)을 쌍으로 설정할 수 있다.

##### 1.2.2.2 Ollama 설치와 모델 다운받기
로컬 PC에서 오픈소스 AI 모델을 구동하기 위해 Ollama와 연동할 수 있다.
1. [Ollama 공식 홈페이지](https://ollama.com)에서 Windows 버전 설치 파일을 다운로드하여 설치한다.
2. 터미널을 실행한 후 연동할 모델을 다운로드한다.
   ```powershell
   ollama run llama3
   ```
3. StyledMD API 설정 창의 Ollama API 주소 영역에 로컬 Ollama 주소(예: `http://localhost:11434`)를 기입하고, 사용하려는 모델명(예: `llama3`)을 기입하여 연동한다.

##### 1.2.2.3 선택된 텍스트를 인공지능으로 다듬기
마크다운 편집 중에 단어 혹은 본문 문장을 블록 지정한 뒤 `Ctrl+.` 단축키를 눌러 AI 채팅 팝업창을 연다.
* **퀵 프롬프트 단축 버튼**:
  * **뜻**: 블록 지정된 텍스트의 뜻을 명확히 정의하여 설명한다.
  * **요약**: 지정된 영역의 주요 요점을 항목별로 요약하여 출력한다.
  * **영어로 / 한글로**: 지정 영역을 대상 언어로 실시간 번역한다.
  * **개념확장**: 입력된 개념을 교육하기 위한 목차를 생성한다.
  * **개념설명**: 입력된 개념에 대해 상세한 해설 본문을 자동으로 작성한다.
  * **전개예측**: 기존 텍스트의 다음 전개 내용을 논리적으로 예측하여 확장 본문을 생성한다.
* 생성된 결과물은 하단의 **본문 삽입** 단추를 눌러 에디터 내 커서 위치에 즉시 삽입할 수 있다.

## 2. 앱을 빌드하기 위한 과정

개발 환경 및 소스 빌드를 진행하기 위해 아래 과정을 차례대로 거친다.

### 2.1 github에서 소스받기
```powershell
git clone <repository-url>
cd <repository-directory>
```

### 2.2 Go설치
1. [Go 공식 다운로드 페이지](https://go.dev/dl/)로 이동하여 Windows 인스톨러(`.msi`)를 다운로드한 후 설치한다. (Go 1.24 이상 필요)
2. 터미널에서 아래 명령을 수행하여 설치 여부 및 버전을 검증한다.
   ```powershell
   go version
   ```

### 2.3 go mod init StyledMD
이 프로젝트는 이미 `styledmd` 모듈로 초기화되어 있으므로, 새로운 프로젝트에서 처음부터 시작하는 경우에만 아래 명령을 통해 Go 모듈을 생성한다.
```powershell
go mod init styledmd
```

### 2.4 외부 라이브러리들을 설치하기
소스 코드가 완전히 확보된 상태에서 아래 명령을 실행한다. `go mod tidy`는 소스 코드 내의 임포트(import) 문을 분석하여 필요한 외부 의존성 패키지를 자동으로 탐색 및 설치하므로, 개별적인 `go get` 명령을 수동으로 수행할 필요가 없다.
```powershell
# 소스 코드가 준비된 후 실행하여 의존성 라이브러리 자동 분석 및 다운로드
go mod tidy

# Windows 실행 파일 정보 및 아이콘 추가를 위한 go-winres 도구 전역 설치
go install github.com/tc-hib/go-winres@latest
```

## 3. 빌드

Windows 환경에 특화된 컴파일 환경을 구성하여 최종 단일 실행 파일로 추출한다.

```powershell
# 프로젝트 루트 경로에 위치한 build.ps1 스크립트 실행
.\build.ps1
```

* **리소스 컴파일**: `winres/winres.json`에 등록된 창 아이콘 및 메타데이터 정보를 컴파일하여 바이너리에 합병한다.
* **콘솔 숨김 빌드**: `-H windowsgui` 플래그를 통해 더블클릭 실행 시 배경 백그라운드 콘솔 창이 뜨지 않도록 처리한다.
* **정적 웹 자원 내장**: `frontend/` 경로의 리소스 파일(HTML, CSS, JS 등)은 Go 1.16+ `//go:embed` 지시어를 활용해 최종 `StyledMD.exe` 바이너리 단일 파일 내에 물리적으로 바인딩된다.

## 4. 추가 기능 및 정보 (기타 설정)

### 4.1 단축키 일람

| 단축키 | 작동 기능 |
|---|---|
| `Ctrl + N` | 새 문서 작성 |
| `Ctrl + O` | 로컬 마크다운 파일 열기 |
| `Ctrl + S` | 현재 파일 저장 |
| `Ctrl + Shift + S` | 다른 이름으로 파일 저장 |
| `Ctrl + .` | AI 채팅창 호출 및 종료 |
| `Enter` | AI 프롬프트 전송 |
| `Shift + Enter` | AI 프롬프트 입력창 내부 줄바꿈 |

### 4.2 시스템 프롬프트 설정 (UserInstructions.md)
* 실행 파일인 `StyledMD.exe`와 동일한 디렉토리에 `UserInstructions.md` 파일을 생성하여 사용자만의 커스텀 지침(답변 톤, 어조, 핵심 제약 사항 등)을 AI에 영구적으로 주입할 수 있다.
* 해당 파일이 부재할 경우 내장된 디폴트 에이전트 설정값을 사용한다.
